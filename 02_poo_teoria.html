Els 4 pilars de la POO

Cau a cada PAF. Aprèn-los amb un exemple, no com a definicions.

1️⃣ Encapsulació

Amagar les dades internes d'una classe darrere de mètodes públics (getters/setters).

public class Cientific {
    private String dni;          // privat: ningú hi accedeix directament

    public String getDni() {   // accés controlat
        return dni;
    }
}

Per a què serveix? Si demà canvies el tipus de dni a long, només has de tocar la classe Cientific, no totes les classes que la fan servir.

2️⃣ Herència

Una classe filla rep els atributs i mètodes de la pare.

public class DirectorEstacio extends Cientific {
    private double pressupost;
    // hereta dni, nom, especialitat... de Cientific
}

Patró: "A és un B" → herència. Director és un científic amb extres.

3️⃣ Polimorfisme

El mateix mètode es comporta diferent segons el tipus real de l'objecte.

Cientific c = new DirectorEstacio();
c.treballar();  // crida la versió de DirectorEstacio, no la de Cientific

Truc per recordar: "poli-morf" = "moltes formes". Mateixa interfície, diferents implementacions.

4️⃣ Abstracció

Quedar-se només amb el que importa, ignorar els detalls irrellevants.

Una classe abstracta o una interfície defineix què s'ha de fer, però no com.

public abstract class Equipament {
    public abstract void arrencar();  // què, no com
}

⚠️ Exemple PAF

Si una classe Campanya té activar() i la filla CampanyaRRSS el redefineix exactament igual (mateixa signatura), és sobreescriptura.

Si hi hagués activar(int data) a la mateixa classe, seria sobrecàrrega.

Compilació, interpretació i JVM

El procés: codi font → execució

codi font (.java)
       │
       │ javac = COMPILACIÓ
       ▼
bytecode (.class)
       │
       │ JVM = INTERPRETACIÓ / JIT
       ▼
codi màquina (s'executa)

Diferències clau

ProcésQuè faEina
Compilació Tradueix codi font a un format que la màquina o la VM entén Compilador (javac, gcc)
Enllaçament (linking) Combina diversos mòduls compilats + biblioteques en un executable Enllaçador (linker)
Interpretació Llegeix codi font o bytecode i l'executa línia a línia Intèrpret (JVM, Python)
Execució El processador (real o virtual) corre les instruccions CPU o VM
JVM (Java Virtual Machine)

El gran avantatge de Java: "Write Once, Run Anywhere".

  • Compiles una vegada a bytecode (.class).
  • El bytecode el pot executar qualsevol JVM (Windows, Mac, Linux...).
  • Portabilitat = no has de recompilar per a cada SO.
⚠️ Trampa de la PAF

"La JVM permet principalment..." NO és "executar més ràpid que codi natiu" (això és fals — el codi natiu sempre és més ràpid). La resposta és la portabilitat.

Modularitat i programació estructurada

Modularitat

Dividir un programa en parts independents i funcionalment tancades.

Avantatges
  • Reutilització: el mateix mòdul serveix per a múltiples projectes
  • Manteniment: arreglar un bug només afecta un mòdul
  • Treball en equip: cada persona pot tocar el seu mòdul
  • Testing: es proven mòduls per separat

Teorema de l'estructura (Böhm-Jacopini)

Qualsevol algorisme es pot construir amb només 3 estructures de control:

  1. Seqüència — una instrucció després d'una altra
  2. Selecció — if / else / switch
  3. Iteració — while / for / do-while
❌ El que NO entra al teorema

GOTO (salts incondicionals). La programació estructurada va sorgir precisament per eliminar els GOTOs, perquè feien el codi inseguible ("codi espagueti").

Conceptes relacionats

ConcepteSignificat
CohesióCom de relacionades estan les responsabilitats dins un mòdul. Alta cohesió = bo.
AcoblamentCom de dependents són els mòduls entre si. Baix acoblament = bo.
JerarquiaOrganització en nivells (mòduls que en contenen d'altres).

Metodologies de desenvolupament — Mètrica 3

Metodologia espanyola per al desenvolupament de sistemes d'informació. Les seves 4 fases cauen a la PAF:

1. Estudi de viabilitat (EVS)

"Val la pena fer aquest projecte?" Anàlisi de cost-benefici, possibilitats tècniques.

2. Anàlisi del sistema d'informació (ASI)

"Què ha de fer el sistema?" Recull de requisits, casos d'ús, model conceptual.

Aquí es fa l'especificació detallada de QUÈ.

3. Disseny del sistema d'informació (DSI)

"Com ho farem?" Arquitectura, disseny de classes, base de dades, interfícies.

Aquí es fa l'especificació detallada de COM.

4. Construcció i implantació (CSI / IAS)

Codificació, proves, posada en producció.

5. Manteniment dels sistemes d'informació (MSI)

Correctiu, evolutiu, adaptatiu, perfectiu.

⚠️ Trampa clàssica

"En quina fase es realitza l'especificació detallada del sistema?"

Tothom contesta "Anàlisi" perquè sona lògic. Però la resposta correcta segons l'EAC2 és "Disseny del sistema d'informació" — l'anàlisi diu QUÈ ha de fer, el disseny detalla COM.

Atenció: hi ha PAFs on s'accepta "Anàlisi" si la pregunta s'enfoca cap als requisits. Llegeix bé l'enunciat.

Quiz — POO i teoria (12 preguntes)

Quiz d'autoavaluació (no compta a la puntuació PAF).

1. Quin pilar de la POO permet que una classe filla rebi atributs i mètodes de la pare?
Herència = "és un". Una classe filla hereta tot el que té la pare. Polimorfisme és una conseqüència útil de l'herència, però el mecanisme bàsic és l'herència.
2. Si Campanya té el mètode activar() i CampanyaRRSS el redefineix amb la mateixa signatura, és:
Sobreescriptura: mateixa signatura, classe filla redefineix el comportament. Sobrecàrrega seria a la mateixa classe amb paràmetres diferents.
3. El procés de traducció del codi font a un format que el processador entén (directament o via VM) s'anomena:
Compilació = traducció. L'enllaçament és el pas posterior que junta mòduls. Execució és córrer el codi ja traduït.
4. La JVM permet principalment:
"Write once, run anywhere". El bytecode compila una vegada i corre a qualsevol SO amb JVM.
5. Un llenguatge que usa codis mnemotècnics (MOV, ADD, JMP) és de:
Assemblador = 2G. La 1G és el binari pur. SQL és 4G, Prolog és 5G.
6. Dividir un programa en parts independents i tancades funcionalment es coneix com:
Modularitat. Jerarquia és l'organització en nivells però no l'acte de dividir.
7. Quin paradigma es basa en regles lògiques per deduir conclusions?
El paradigma lògic (Prolog) parteix de fets i regles, dedueix conclusions. El funcional es basa en funcions matemàtiques pures.
8. A Mètrica 3, l'especificació detallada del sistema es realitza a:
Aquesta és la resposta segons la solució oficial de l'EAC2. Anàlisi = QUÈ ha de fer. Disseny = COM es farà (especificació detallada). Atenció: és una pregunta amb matís — llegeix l'enunciat de la PAF amb cura.
9. Quin element NO forma part de la programació estructurada segons el teorema de l'estructura?
Böhm-Jacopini: qualsevol algorisme es pot fer amb seqüència, selecció i iteració. El GOTO és justament el que la prog. estructurada va eliminar.
10. Conjunt de codi predefinit que facilita la tasca de codificació:
Biblioteca = codi reusable. Si fos un framework, el codi et cridaria a tu en lloc de tu cridar-lo a ell.
11. L'encapsulació serveix principalment per:
Encapsulació = ocultació + accés controlat. Si demà canvies com s'emmagatzemen les dades, només toques la classe.
12. La diferència principal entre sobrecàrrega i sobreescriptura és:
Sobrecàrrega = mateix nom, diferent llista de paràmetres (mateixa classe). Sobreescriptura = mateixa signatura, classe filla redefineix. L'opció c és al revés: sobrecàrrega és en compilació, sobreescriptura en execució.
