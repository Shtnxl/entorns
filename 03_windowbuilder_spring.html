El que entra a la PAF d'aquest bloc

A la PAF no has d'escriure codi de WindowBuilder ni Spring. Has de saber interpretar-lo: identificar què fa cada classe, mètode, anotació, etc.

El que t'han de preguntar
  • "Aquest mètode actionPerformed què fa?"
  • "Quin component representa JComboBox?"
  • "Què fa l'anotació @GetMapping?"
  • "Per què el botó queda deshabilitat?"
  • "Quin endpoint URL respon aquest controlador?"

Propietats que sempre cauen

setEnabled / setEditable
txtAntiguitat.setEditable(false); // no es pot escriure dins
btnValidar.setEnabled(false);    // no es pot clicar

Diferència:

  • setEditable només existeix a camps de text — bloqueja l'escriptura però es pot seleccionar/copiar.
  • setEnabled existeix a tots els components — bloqueja interacció i es veu en gris.
Tooltips (text d'ajut)
txtAntiguitat.setToolTipText("Nombre enter entre 0 i 150");

Apareix quan l'usuari hi passa el ratolí per sobre. L'enunciat EAC2 ho demanava explícitament per a cada quadre de text editable.

⚠️ Seleccionar tot + focus (truc per errors)

Quan validació falla, has de ressaltar el camp incorrecte:

txtAntiguitat.requestFocus();
txtAntiguitat.selectAll();

Ordre crític: primer focus, després selecció. Si ho fas al revés, el focus deselecciona la selecció.

Llegir codi generat per WindowBuilder

Estructura típica d'una classe Swing

public class PressupostMarketing {

    private JFrame frame;
    private JTextField txtAntiguitat;
    private JComboBox<String> comboPack;
    private JButton btnValidar;
    private JTextArea txtErrors;

    // 1. Mètode main: arrenca l'app
    public static void main(String[] args) {
        EventQueue.invokeLater(() -> {
            try {
                PressupostMarketing window = new PressupostMarketing();
                window.frame.setVisible(true);
            } catch (Exception e) {
                e.printStackTrace();
            }
        });
    }

    // 2. Constructor: crida initialize()
    public PressupostMarketing() {
        initialize();
    }

    // 3. Crea tota la GUI
    private void initialize() {
        frame = new JFrame();
        frame.setTitle("Pressupost - ElRahali");
        frame.setBounds(100, 100, 600, 500);

        txtAntiguitat = new JTextField();
        txtAntiguitat.setToolTipText("Anys (0-150)");

        btnValidar = new JButton("Validar");
        btnValidar.addActionListener(e -> validar());
    }

    // 4. Listener: lògica del botó
    private void validar() {
        try {
            int anys = Integer.parseInt(txtAntiguitat.getText());
            if (anys < 0 || anys > 150) {
                txtErrors.setText("Antiguitat fora de rang");
                txtAntiguitat.requestFocus();
                txtAntiguitat.selectAll();
                return;
            }
            // Dades correctes: deshabilita Dades Client, habilita Campanya
            txtAntiguitat.setEnabled(false);
            comboGestioXXSS.setEnabled(true);
        } catch (NumberFormatException ex) {
            txtErrors.setText("L'antiguitat no és un nombre vàlid");
        }
    }
}

Què mirar per "interpretar" aquest codi

1. Identificar els 4 blocs
  1. main(): arrenca l'aplicació amb EventQueue.invokeLater (per al thread d'Swing).
  2. Constructor: crida initialize().
  3. initialize(): crea i configura tots els components visuals.
  4. Listeners: la lògica que reacciona als clics i canvis.
2. Identificar listeners

Hi ha 2 sintaxis:

// Clàssica (classe anònima)
btnValidar.addActionListener(new ActionListener() {
    public void actionPerformed(ActionEvent e) {
        validar();
    }
});

// Lambda (moderna, més curta)
btnValidar.addActionListener(e -> validar());

Totes dues fan el mateix. A la PAF, si veus e ->, és una lambda; si veus new ActionListener(), és classe anònima.

⚠️ Comprovacions de validació típiques

Quan al codi veus aquest patró:

if (combo.getSelectedIndex() == -1) { ... }

Significa "l'usuari no ha seleccionat cap element del desplegable". -1 indica "res seleccionat".

if (camp.getText().isEmpty()) { ... }

Significa "el camp està buit".

Llegir codi Spring Boot

Mini-recordatori de l'EAC2 (Exercici 12)

El que vas fer: un controlador Controlador amb dos endpoints / i /quintuple que multiplica per 5 un nombre rebut.

Anotacions clau

AnotacióQuè fa
@SpringBootApplicationMarca la classe d'arrencada de Spring Boot
@ControllerMarca una classe com a controlador web (retorna noms de vistes)
@RestControllerIgual però retorna directament dades JSON/text
@GetMapping("/url")Aquest mètode respon a peticions GET a la URL
@PostMapping("/url")Aquest mètode respon a peticions POST
@RequestParamCaptura un paràmetre de la URL (?valor=5)

Exemple complet (com el de l'EAC2)

package com.example.web;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;

@SpringBootApplication
@Controller
public class Controlador {

    public static void main(String[] args) {
        SpringApplication.run(Controlador.class, args);
    }

    @GetMapping("/")
    public String inici() {
        return "formulari";  // nom de la vista (formulari.html)
    }

    @GetMapping("/quintuple")
    public String quintuple(@RequestParam("valor") int valor,
                            Model model) {
        int resultat = valor * 5;
        model.addAttribute("resultat", resultat);
        return "resultat";  // vista resultat.html
    }
}

Com llegir aquest codi a la PAF

Pregunta: "Quina URL respon aquest controlador?"

Mira els @GetMapping: / i /quintuple. La URL completa seria http://localhost:8080/quintuple?valor=10.

Pregunta: "Què retorna el mètode quintuple()?"

Retorna l'String "resultat", que no és el resultat numèric! És el nom de la vista (la plantilla HTML que es renderitzarà). El resultat numèric es passa al model amb model.addAttribute.

Pregunta: "Per què cal @RequestParam?"

Indica que el valor ve de la URL com a paràmetre (?valor=10). Sense l'anotació, Spring no sabria d'on treure el valor.

Exercicis de lectura (estil PAF)

Llegeix cada fragment i intenta respondre abans d'obrir el desplegable.

📖 Exercici 1 — Què fa aquest listener? 
btnEsborrarTot.addActionListener(e -> {
    txtAntiguitat.setText("");
    txtAntiguitat.setEnabled(true);
    comboPack.setSelectedIndex(-1);
    comboGestioXXSS.setEnabled(false);
    txtErrors.setText("");
});

Resposta: Quan es clica el botó "Esborrar tot": (1) buida el camp d'antiguitat, (2) el torna a habilitar, (3) desselecciona el combo del pack, (4) deshabilita el combo de gestió de XXSS (perquè ha de tornar a la fase de Dades Client), (5) neteja la caixa d'errors. És el reset complet de l'aplicació.

📖 Exercici 2 — Què fa aquest fragment d'Spring? 
@GetMapping("/calcula/{a}/{b}")
public String calcula(@PathVariable int a,
                      @PathVariable int b,
                      Model model) {
    model.addAttribute("suma", a + b);
    return "calcul";
}

Resposta: Respon a URLs com /calcula/3/7. @PathVariable captura els fragments de la URL (a=3, b=7). Posa la suma al model i retorna la vista calcul.html. Diferent de @RequestParam, que faria servir /calcula?a=3&b=7.

📖 Exercici 3 — Què passa quan l'usuari clica "Validar"? 
private void validar() {
    int idxPack = comboPack.getSelectedIndex();
    if (idxPack == -1) {
        txtErrors.setText("Selecciona un pack");
        comboPack.requestFocus();
        return;
    }
    try {
        int anys = Integer.parseInt(txtAntiguitat.getText());
        if (anys < 0 || anys > 150) {
            throw new NumberFormatException();
        }
        txtErrors.setText("");
        seccionCampanya.setEnabled(true);
    } catch (NumberFormatException ex) {
        txtErrors.setText("Antiguitat invàlida (0-150)");
        txtAntiguitat.requestFocus();
        txtAntiguitat.selectAll();
    }
}

Resposta:

  1. Si no hi ha pack seleccionat (index = -1) → error, focus al combo, surt.
  2. Intenta parsejar l'antiguitat com a enter.
  3. Si està fora de 0-150 o no és nombre → catch llança error, focus + selectAll al camp.
  4. Si tot va bé → neteja errors i habilita la secció Campanya.
📖 Exercici 4 — Què té de particular aquesta línia? 
SpringApplication.run(Controlador.class, args);

Resposta: Aquesta línia arrenca tota l'aplicació Spring Boot:

  • Inicialitza el context de Spring (registra controladors, beans...).
  • Arrenca el servidor Tomcat embegut (per defecte al port 8080).
  • Activa l'auto-configuració de @SpringBootApplication.

Per això a Spring Boot no necessites instal·lar Tomcat per separat com en l'enfocament tradicional amb servlets.

📖 Exercici 5 — Per què aquest tooltip és important? 
txtDescompte.setToolTipText("Euros a descomptar (0 fins preu base del pack)");

Resposta: Els tooltips compleixen l'obligació de l'enunciat de l'EAC2: "Cal posar un text d'ajut a cada quadre de text editable". Indica a l'usuari què espera la validació (un valor entre 0 i el preu del pack). Si la PAF mostra codi sense tooltip i pregunta "què falta?", podria ser aquesta la resposta.

Quiz — Interpretació de codi (10 preguntes)

1. Què fa la línia combo.getSelectedIndex() == -1 en un if?
getSelectedIndex() retorna l'índex (0-based) de l'element seleccionat. -1 significa "no hi ha res seleccionat".
2. txt.requestFocus() i txt.selectAll() serveixen per:
És el patró típic per ressaltar un camp amb dada errònia: focus + selecció permeten que l'usuari escrigui directament i sobreescrigui el valor.
3. L'anotació @GetMapping("/quintuple") indica que:
GetMapping = GET. PostMapping = POST. Es pot accedir directament des de la barra d'adreces del navegador (que sempre fa GET).
4. Diferència entre @RequestParam i @PathVariable:
/quintuple?valor=5 → @RequestParam · /quintuple/5 → @PathVariable.
5. Quan btn.setEnabled(false):
Desactivar (enabled=false) no oculta, només bloqueja la interacció. Per ocultar caldria setVisible(false).
6. model.addAttribute("resultat", n); en un controlador Spring serveix per:
El Model és el "transportador" de dades entre el controlador i la plantilla (Thymeleaf, JSP...). A la plantilla apareixerà com a variable ${resultat}.
7. SpringApplication.run(...) a Spring Boot:
Spring Boot porta Tomcat embegut. Per això no cal instal·lar res a part — l'aplicació és executable.
8. En private JComboBox<String> comboPack;, el <String> indica:
És un generic Java. JComboBox<String> conté Strings. Si fos JComboBox<Integer>, contindria enters.
9. Què passa amb el següent codi si l'usuari introdueix "abc"?
int n = Integer.parseInt(camp.getText());
parseInt llança NumberFormatException si la cadena no és parsejable. Per això sempre s'embolica amb un try/catch a les validacions de l'EAC2.
10. EventQueue.invokeLater(...) al main() serveix per:
Swing no és thread-safe. Tots els canvis a la GUI han de fer-se a l'Event Dispatch Thread. invokeLater hi posa el codi en cua.
